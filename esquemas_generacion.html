Representaciones de código intermedio

Durante la fase de síntesis y análisis de código intermedio de un compilador se pueden observar dos partes un front-end que se encarga de analizar el código fuente y generar un código intermedio para que después un back-end genere el código destino o código maquina.

Si se tiene una buena representación de código intermedio podemos construir un compilador para el lenguaje i y la maquina j mediante la comprobación del front-end para el lenguaje i y el back-end para la maquina j, esto nos permitiría crear un conjunto de compiladores ahorrando esfuerzo de hacerlo uno a uno, podríamos construir m x n compiladores con solo escribir m front-end y n back-end.

Para la representación de código intermedio existen varias opciones como los arboles sintácticos y el código de tres direcciones (triplos y cuadruplos). El código de tres direcciones recibe ese nombre de las instrucciones del tipo x = y operador z , donde se usan tres direcciones de memoria, una para x que es donde se almacena el resultado y dos para y y z. Durante la traducción de código fuente a código intermedio de un compilador se puede generar diferentes secuencias de representaciones intermedias. Las representaciones de alto nivel están mas cerca del código fuente mientras que las de bajo nivel están mas cerca del lenguaje maquina.

Las representaciones de bajo nivel son adecuadas par la asignación de registros de memoria y la selección de instrucciones.

Variantes de los arboles sintácticos.

Grafos Dirigidos Aciclicos (GDA).

Similar a los arboles sintácticos los GDA se componen de hojas correspondientes a los operandos y nodos interiores correspondientes a los operadores, la diferencia radica en que un nodo puede tener mas de un padre, es decir ya no es un árbol binario, por ende un GDA representa las expresiones de una forma mas breve.
